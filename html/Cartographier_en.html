{oA.Lineage}

Mapping an Information System

Information systems are so complex that teams are no longer able to effectively navigate the flows to carry out the desired investigations. In recent times, domain-oriented Data Mesh architectures have accelerated this state of affairs.

Our multi-technology data lineage, coupled with the analysis of data usage, makes it possible to map systems precisely and continuously.

Faire du Data Lineage dans les Dataflows

Au départ de n’importe quel « data point » du Système d’Information (un champ d’une table, une table, un fichier, un schéma, etc.), {openAudit} permet d'appréhender son origine, son devenir et ses usages grâce au data lineage technique, multi technologique. Le process est dynamique, opéré en mode delta, et sans rupture : {openAudit} traite les vues, les procédures dynamiques, etc.

Une analyse d’impact de bout en bout

{openAudit} propose une vue qui permet de connaître instantanément l’ensemble des usages d’un data point dans la couche de data visualisation, en faisant abstraction de toutes les transformations.

Une cartographie complète et instantanée

{openAudit} permet, à partir de n'importe quel data point, d’afficher une cartographie complète du flux amont et aval, depuis les sources opérationnelles jusqu'à l'exposition des données (dataviz, requêtes), c’est à dire leurs usages.

Une data lineage granulaire

{openAudit} permet de suivre de façon progressive le déploiement d’une donnée dans les Systèmes d’Information complexes, on prem’, Cloud, sans rupture. Chaque transformation peut être analysée avec le « drill through ».

Faire du Data Lineage dans la couche de Data visualisation

{openAudit} permet d'appréhender l'ensemble des technologies de data visualisation de l'entreprise sur une interface unique d'analyse d'impact. Le data lineage dans le dashboard peut être déclenché pour mettre en lumière les nombreuses règles de gestion qui y sont définies.

Mise en lumière des règles de gestion

{openAudit} opère un data lineage dans les expressions, dans les variables, etc., pour expliciter précisément les règles de gestion qui sous-tendent le calcul d’un indicateur d’un dashboard

Un sourcing automatique des indicateurs du dashboard

{openAudit} analyse le contenu des vues pour identifier les champs physiques qui sont en source des données d’un dashboard, même si les vues sont empilées.

Une grille d’analyse d’impact combinée

{openAudit} combine les analyses des différentes technologies de dataviz de l’entreprise dans une même grille, ce qui permet au métier et à l'IT de réaliser des analyses d'impact exhaustives.
